<DB fail 유형>
1.Statement fail
-DB에 발생된 명령문(SQL구문)이 정상적으로 실행되지 않고 오류가 발생된 경우
-원인 : 오타, 문법 오류, 권한 부족, 공간 부족 등
-해결 : 명령문 재작성, 권한 부여, 공간 부여 등

2.User proc fail
-DB에 접속된 User가 비정상적으로 접속을 종료한 경우
-원인 : User가 비정상적으로 접속을 종료함, DBA에 의해서 Kill이 된 경우,
       Client PC가 다운된 경우, Network 불안정해서 접속이 끊어진 경우 등
-해결 : 재접속하면 됨(PMON이 트랜잭션 롤백 및 Lock 해제함)

3.Network fail
-DB가 정상적으로 켜져 있는데 DB에 접속이 안되는 경우
-원인 : 리스너가 꺼져있는 경우, Network가 불안정한 경우 등
-해결 : 리스너 켜고 DB 재시작하면 됨. Network 안정화 작업 등

4.User error(사용자 실수)
-DB에 발생된 명령문(SQL구문)이 정상적으로 실행되었으나 잘못된 작업을 진행한 경우
-원인 : 잘못된 DML, 잘못된 DDL 등
-해결 : 트랜잭션 진행 중이면 rollback, 
       commit을 이미 실행했다면 반대의 작업을 다시 수행해야함,
       테이블을 실수로 삭제했다면 휴지통(recyclebin)으로부터 flashback함

5.Instnace fail
-DB가 비정상적으로 종료된 경우
-원인 : DBA가 DB를 비정상종료(abort)함.
       Server PC가 다운되거나 정전 발생함.
       Instance 내 필수 구성요소 중 하나가 오류가 발생된 경우 등
-해결 : DB 재시작하면 됨(DB 재시작하는 동안에 SMON이 DB 동기화 맞춰줌)

6.Media fail
-DB를 구성하는 필수 물리적인 파일 중 하나가 손실/손상된 경우
 (Datafile, Controlfile, Redologfile)
-원인 : 운영체제에서 Datafile, Controlfile, Redologfile을 실수로 삭제한 경우
-해결 : Datafile - 백업파일로부터 복구 작업 진행해야함.
       Controlfile - 다중화된 파일로부터 복구 작업 진행해야함.
       Redologfile - 다중화된 파일로부터 복구 작업 진행해야함.
