<<Oracle Database 전체구조>>
<Database>
1.Datafiles
-실제 Data가 저장된 곳
-Data Dictionary가 저장된 곳
-Datafile 정보 조회 : v$datafile, v$tempfile

2.Controlfile
-DB의 무결성을 유지/관리할 수 있는 DB의 모든 동기화 정보가 기록된 곳
 (DB의 논리적/물리적인 구조 정보, 마지막 작업 번호 등...)
-DB당 하나만 존재하면 됨
-다중화 기능 제공됨(최대 8개까지 다중화 가능함, 권장사항은 3개)
-Controlfile 정보 조회 : v$controlfile

3.(Online) Redologfiles
-DB에서 발생된 변경 작업의 로그정보(Redo data)가 기록된 곳
-주목적 : Datafile Recovery(복구)
-DB당 최소 두개이상 존재하면 됨
-순환형 관리됨, 다중화 기능 제공됨
-DB당 Redo log Group(논리적인 구조)이 2개이상 존재해야함
-Redo log Group에는 Redo log Member(file, 물리적인 구조)가 1개 이상
 존재해야함
-권장사항 : Group 3개, Member 2개씩
-Redologfile 정보 조회 : v$log, v$logfile

4.Parameterfile
-Instance의 정의정보가 기록된 곳(SGA 할당정보, B/G proc 정보 등)
-DB의 여러 설정 정보가 기록된 곳
-위치 : $ORACLE_HOME/dbs
       (Oracle Software가 설치된 물리적인 경로가 설정된 환경변수)
-이름 : spfileSID.ora 또는 initSID.ora

5.Passwordfile
-DB를 시작/종료할 수 있도록 인증해주는 곳
-위치 : $ORACLE_HOME/dbs
-이름 : orapwSID

6.Archived log files(option)
-Redo log file의 Offline 복사본
-주목적 : Datafile Recovery(복구)
-log mode : Noarchive log mode vs Archive log mode(권장) 중 선택
-log mode 정보 조회 : v$database

<Instance>
(SGA) : 공유가 가능한 메모리 구조
1.Shared Pool : 사전 준비 작업을 하는 메모리 구조
1)Library cache
-최근에 실행된 SQL구문과 실행계획이 저장된 곳
 (대소문자, 공백까지 일치되어야 hit됨)
-LRU 알고리즘으로 관리됨

2)Data dictionary cache
-최근에 사용된 DB 정의정보가 저장된 곳
 (file, object, 권한, 제약조건 등)
-LRU 알고리즘으로 관리됨

2.DB Buffer Cache : 실제 실행 작업을 하는 메모리 구조
-최근에 사용된 Data block이 저장된 곳
-LRU 알고리즘으로 관리됨

3.Redo Log Buffer
-DB에서 발생된 변경작업의 로그정보(Redo data)가 기록되는 곳
-순환형으로 관리됨

(B/G proc)
1.PMON - User proc fail 시 진행하던 트랜잭션 롤백 및 선점하고 있던 자원과 
         Lock을 해제함.
2.SMON - Instance fail(=DB 비정상종료) 후 DB 재시작될때 DB 동기화를 맞춰줌.
3.DBWR - DB buffer cache의 Dirty block(변경된 block)을 Datafile로 기록함.
         체크포인트 발생 시 기록함.
4.LGWR - Redo log buffer의 로그정보(Redo data)를 Redo log file로 기록함.
         커밋(commit)이 발생될때 기록함.
5.CKPT - 체크포인트 발생 시 DBWR에게 알려줌.
         DBWR 기록 후 Datafile 헤더와 Controlfile에 마지막 체크포인트 번호
         갱신함.
6.ARC(option) - Archive log mode인 경우 활성화됨.
		Redo log file에 로그스위치(logswitch) 발생 시 Offline 
                복사본(Archived log file)을 생성함.
